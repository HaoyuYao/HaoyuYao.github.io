Fooling automated surveillance cameras- adversarial patches to attack person detection

发表于 2023-03-14

Essence

Motivation

  1. All the approached to make adverserial attack the works before propose barely contains intra-class variety
    1. the objects like human have intra-class variety which may be harder to attack.
  2. in this work, the author present an approach to generate patches to targets with lots of intra-class variety like person.
  3. Author realize a realtive small patch which is able to hide person from person detective systems.

preliminary

relative works

  1. Adversarial attacks on classification tasks
    1. slightly change the value of several pixels in an image
    2. a gradient sign method made the attacking process more faster.
    3. using hyper-planes to model the border between output classes.
    4. using patches: 1. stickers that can attack in many poses, 2. using a 3D sticker with robustness to different poses and lighting. 3. methods to generate a universarial image
  2. Real-world adversarial attack for face recognition
    1. A pair of glasses the fool the facial recognition systems, which emphasize the practicability.
  3. Real-world adversarial attacks for object detection
    1. for stop signs, using Expection over Transformation(EOT) to build in robustness for poses.
  4. Object detection
    YOLOv2’s principle:
    1. every image is handled to an output grid with a resolution 32 times smaller than the input’s. Each cell in the output contains five predictions(anchor point) with bounding boxes with different aspect ratios. an anchor point has a vector describing the position of the bounding boxes and the possiblities of every class.
    2. ![[Pasted image 20230314155737.png]]

methods

goal :

The goal of this work is create a system that is able to generate printable adversarial patches that can be used to
fool person detectors.

innovation:

the previous work only made real-world adverserial attack on uniform things like stop sign. author wanna do the same thing on person which is more various.

method:

optimisation goal contains 3 parts:

  1. the printability of the patch.
  2. the variation of the image, this should be the smaller the better. our optimiser favours an image with smooth colour transitions and prevents noisy images.
  3. to make the difficulty of detecting max.
    ![[Pasted image 20230314161043.png]]

author tested 3 approached:

  1. minimize the classification probability of person.
  2. minimize the objectness score.
  3. the combination of both.

and they found:

  1. the approach 2 perform best, for the patch it generate is more universal.
  2. the others had each limits because they cause the patch to resemble another class.(if a new detector doesn’t have such a class, the result may sucks)

preparing the training datasets

there’s 4 chanllenges:

  1. people are more various.
  2. people can appear in various contexts.
  3. people can appear in various poses.
  4. patches can be attached to various position on person.
    what the author do:
  5. use real images of different people, instead of doing transformation in a single image of the target object.
  6. use detector to find the position of human in each image, then attach patches on the image under various transformation.
  7. feed the patched images to detector and test the loss function.
  • one side effect is that the author is not limited to annotated datasets. so that it can just apply to any available datasets.

making patched more robust

  • reason: in reality, the various lighting and poses of the patch ask for more robustness of the patch.
  • so author so some transformation on patch before it applied to image:
    • rotate
    • scale up and down
    • random noise
    • various brightness and contrast

results

  • using random noise as control to test the performance of patches.
  • the result tell that minimizing the objectness score is best approach.

defect

  1. the patch only perform well while it attached to the center of human.

future work

  1. robust
  2. transformation

problem

  1. what is an optimisation process?
  2. what is to freeze the weights in:The goal of our optimizer is to minimise the total loss L. During the optimisation process we freeze all weights in the network, and change only the values in the patch. The patch is initialised on random values at the beginning of the process.
  3. the principle of yolov2
  4. what is the different of object loss and class loss. and so of the object and class score.
  5. what is back propagation.
  6. what does this mean? :Through this entire process it is important to note that it has to remain possible to calculate a backwards gradient on all operations all the way towards the patch
